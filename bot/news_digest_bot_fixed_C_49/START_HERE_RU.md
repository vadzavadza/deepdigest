# Старт с нуля — очень маленькими шагами

Если ты читаешь это и думаешь: **"я вообще не понимаю, куда что вставлять"**, это нормально.

Сейчас твоя задача **не запускать всё сразу**.
Сейчас задача только одна:

## Шаг 1

Открой папку проекта `news_digest_bot`.

Внутри ты должен видеть примерно это:
- `app`
- `tests`
- `docker-compose.yml`
- `.env.example`
- `pyproject.toml`

## Шаг 2

Пока **не открывай код и ничего не вставляй внутрь файлов**.

Просто убедись, что папка проекта открывается и все эти файлы лежат внутри.

## Шаг 3

После этого напиши мне в чат:

`готово, папку открыл`

И только после этого мы пойдём к следующему маленькому шагу.


## Что уже можно проверять в этой сборке
1. Создать тему
2. Открыть «Мои темы»
3. Нажать «▶️ Запустить сейчас»
4. Дождаться, пока бот пришлёт найденные новости в личный чат

Если новостей нет, бот пришлёт сервисное сообщение о том, что новых материалов не найдено.


## Что нового в этой сборке

- исправлен тип OpenRouter web search tool на `openrouter:web_search`;
- добавлен fallback на legacy web plugin;
- в ручном запуске теперь видно диагностику по источникам;
- если LLM summary/relevance падает, бот старается не развалиться, а дать упрощённый результат.
